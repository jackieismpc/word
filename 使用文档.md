# 单词对对碰使用文档

## Mac 使用

启动方式二选一：

1. 终端启动

```bash
cd /Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https
./start-word-match.sh
```

2. 双击启动

- 双击 [start-word-match.command](/Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https/start-word-match.command)

启动后：

- 默认地址是 `http://127.0.0.1:12345`
- 如果 `12345` 被占用，会自动换到下一个空闲端口，比如 `12346`
- 启动脚本会直接打印最终实际地址
- 如果浏览器里还停在旧页面，刷新一次即可加载最新界面

停止方式：

- 推荐在终端执行：

```bash
cd /Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https
./stop-word-match.sh
```

- 如果当前服务就是在前台终端里运行，也可以直接按 `Ctrl + C`
- 服务会优雅退出并释放端口

## Windows 使用

Windows 有两种用法。

### 方式一：Windows 本机直接部署

推荐直接使用仓库里新增的 Windows 脚本，不需要手工配置 Node、PowerShell 执行策略或 npm 依赖。

需要准备：

- Windows 自带浏览器或 Chrome / Edge 任一浏览器
- 能访问 `https://nodejs.org`：首次自动安装本地 Node 运行时要用
- 能访问 `https://oss-cdn.tsdanci.com`：一键准备词库和词包要用
- `Tailscale` 仅在你要让其他设备访问这台 Windows 机器时才需要

一键安装：

1. 双击 [install-word-match.cmd](/Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https/install-word-match.cmd)
2. 脚本会自动：
   - 查找系统里已有的 `node`
   - 如果没有或版本低于 `18`，自动从 `nodejs.org` 下载官方 Windows 便携版到 `.runtime/node`
   - 检查是否装了 Tailscale

一键启动：

1. 双击 [start-word-match.cmd](/Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https/start-word-match.cmd)
2. 脚本会自动：
   - 再次查找可用的 `node`
   - 如有必要自动补装本地 Node
   - 自动寻找 `12345` 起的可用端口
   - 启动服务
   - 自动打开浏览器
   - 打印本机地址和 Tailscale 地址

一键停止：

1. 双击 [stop-word-match.cmd](/Users/jackie/Documents/Codex/2026-05-21/plugin-browser-openai-bundled-chrome-https/stop-word-match.cmd)
2. 脚本会优先优雅停止服务，超时后再强制结束

启动后默认地址：

- `http://127.0.0.1:12345`
- 如果 `12345` 被占用，会自动改成 `12346`、`12347` 等下一个空闲端口

运行日志：

- 标准输出：`.runtime/server.stdout.log`
- 错误输出：`.runtime/server.stderr.log`

注意：

- 第一次启动如果 Windows 防火墙弹窗，允许“专用网络”即可
- 如果公司网络屏蔽了 `nodejs.org` 或 `oss-cdn.tsdanci.com`，自动安装或自动下载词库会失败
- 如果你已经自己安装了 `Node.js 18+`，脚本会优先复用系统 Node，不会重复下载

### 方式二：Mac 部署，Windows 通过 Tailscale 使用

这种方式下，Windows 不需要安装 Node。

步骤：

1. 先在 Mac 上启动本项目
2. Mac 和 Windows 都安装并登录 Tailscale
3. 两台设备加入同一个 Tailnet
4. 查看 Mac 启动脚本打印出来的 `Tailscale 地址`
5. 在 Windows 浏览器打开这个地址

示例：

```text
http://100.116.24.4:12345
```

注意：

- 如果默认端口 `12345` 被占用，脚本会自动换端口
- Windows 端要使用脚本最终打印出来的那个完整地址
- Windows 端直接用浏览器打开即可，不需要安装 Node 或其他开发环境
