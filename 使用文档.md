# 单词对对碰使用文档

本文档说明如何使用项目自带脚本完成安装、启动、停止，以及如何准备离线运行所需的 Node.js。

## 通用说明

- 项目启动服务时不依赖 `npm`、`pnpm` 或额外 `node_modules`
- 安装、启动、停止脚本默认都在项目根目录执行
- 服务默认监听端口 `12345`
- 如果 `12345` 被占用，脚本会自动尝试后续端口
- 运行日志默认写入 `.runtime/server.stdout.log` 和 `.runtime/server.stderr.log`
- 运行状态文件默认写入 `.runtime/server.pid` 和 `.runtime/last-port`

## 离线资源准备

如果目标机器不能稳定联网，建议先在一台可联网的机器上准备好离线 Node.js 包，再把整个项目目录复制过去。

### Windows 离线包

在可联网的 Mac / Linux 机器上执行：

```bash
chmod +x ./prepare-word-match-offline-assets.sh
./prepare-word-match-offline-assets.sh x64
```

如果目标 Windows 是 ARM 设备，可以执行：

```bash
./prepare-word-match-offline-assets.sh arm64
```

准备完成后，离线包会放在：

```text
offline-assets/node/
```

Windows 当前推荐使用的离线包文件名：

- `node-v20.12.2-win-x64.zip`
- `node-v20.12.2-win-arm64.zip`
- `node-v20.12.2-win-x86.zip`

### Mac 离线包

Mac ARM 当前已实测可用的离线包：

- `offline-assets/node/node-v20.12.2-darwin-arm64.tar.gz`

Mac Intel 推荐文件名：

- `offline-assets/node/node-v20.12.2-darwin-x64.tar.gz`

## Mac

### 安装

进入项目目录后执行：

```bash
chmod +x ./install-word-match.sh ./start-word-match.sh ./stop-word-match.sh
./install-word-match.sh
```

安装脚本的默认优先级：

1. `.runtime/node` 中已安装的本地运行时
2. `offline-assets/node` 中的离线压缩包
3. 系统里的兼容 Node.js
4. 显式允许的在线下载

默认不会联网下载。只有显式设置下面这个环境变量后，才允许在线下载：

```bash
export WORD_MATCH_ALLOW_ONLINE_DOWNLOAD=1
```

### 启动

```bash
./start-word-match.sh
```

启动成功后，终端通常会显示：

- 实际使用的 Node 路径
- 本地访问地址，例如 `http://127.0.0.1:12345`
- 日志文件路径

### 停止

```bash
./stop-word-match.sh
```

### 说明

- 当前已实测可用的 Mac ARM 离线包为 `offline-assets/node/node-v20.12.2-darwin-arm64.tar.gz`
- 如果离线包校验失败，脚本会自动回退到系统里的兼容 Node.js，而不会卡死
- 如果服务已经在运行，再次执行 `./start-word-match.sh` 会直接输出当前访问地址，而不是重复启动

```bash
chmod +x ./install-word-match.sh ./start-word-match.sh ./stop-word-match.sh
./install-word-match.sh
```

### 启动

```bash
./start-word-match.sh
```

### 停止

```bash
./stop-word-match.sh
```

## Windows

### 方案一：Windows 原生脚本

这是 Windows 的默认推荐方案，适合离线环境。

#### 第一步：准备项目目录

如果目标 Windows 不能访问外网，建议先在一台可联网的 Mac / Linux 机器上准备好 `offline-assets/node` 中的离线包，然后把整个项目目录原样复制到 Windows。

复制到 Windows 后，请确保下面这些文件都在项目根目录：

- `install-word-match.cmd`
- `start-word-match.cmd`
- `stop-word-match.cmd`
- `offline-assets/node/node-v20.12.2-win-x64.zip` 或对应架构版本

#### 第二步：安装

在资源管理器中双击：

1. `install-word-match.cmd`

安装脚本的默认优先级：

1. `.runtime\node` 中已安装的本地运行时
2. `offline-assets\node` 中的离线 zip 包
3. 系统里的兼容 Node.js
4. 显式允许的在线下载

默认不会联网下载。只有显式设置下面这个环境变量后，才允许在线下载：

```powershell
$env:WORD_MATCH_ALLOW_ONLINE_DOWNLOAD = "1"
```

安装成功后，脚本会输出：

- 实际使用的 Node 路径
- Node 版本
- 启动和停止脚本名称

#### 第三步：启动

在资源管理器中双击：

1. `start-word-match.cmd`

启动成功后，脚本会输出：

- 本地访问地址，例如 `http://127.0.0.1:12345`
- 如果端口被占用，脚本自动切换后的新端口
- 日志文件路径

如果服务已经在运行，再次双击 `start-word-match.cmd`，脚本会直接提示当前访问地址，不会重复启动。

#### 第四步：停止

在资源管理器中双击：

1. `stop-word-match.cmd`

停止脚本会先尝试优雅结束进程，如果服务没有及时退出，再自动强制结束。

### Windows 原生脚本说明

- 当前 Windows 离线安装默认优先使用本地 Node，不需要联网
- 只有在你显式设置 `WORD_MATCH_ALLOW_ONLINE_DOWNLOAD=1` 时，脚本才会尝试访问 `nodejs.org`
- 启动脚本中的健康检查只访问本机 `127.0.0.1`，不访问外网
- `Tailscale` 仅在系统里已经安装时读取本地信息，不会自动下载安装
- 本项目不依赖 npm、pnpm 或额外 `node_modules`

### 方案二：Windows WSL2 Ubuntu 保底方案

这是联网保底方案，不适用于完全离线的 Windows 环境。

默认情况下，WSL 方案是禁用的。只有你明确允许联网时，才应该使用它。

如果你确实要启用 WSL 联网方案，请先设置：

```powershell
$env:WORD_MATCH_ALLOW_WSL_ONLINE = "1"
```

然后按顺序双击：

1. `install-word-match-wsl.cmd`
2. `start-word-match-wsl.cmd`

停止服务时双击：

1. `stop-word-match-wsl.cmd`

### WSL 方案说明

- `install-word-match-wsl.cmd` 会自动启用 WSL2、安装 Ubuntu，并在 Ubuntu 中准备运行环境
- 该流程包含 `wsl --install`、`apt-get update`、`apt-get install`
- 如果 Windows 提示需要重启，请重启后再次运行 `install-word-match-wsl.cmd`
- 如果你的目标是完全离线安装，请不要使用 WSL 方案

## 常见问题

### 1. 安装脚本会不会自动联网？

默认不会。

- Mac / Ubuntu：只有显式设置 `WORD_MATCH_ALLOW_ONLINE_DOWNLOAD=1` 才允许在线下载 Node.js
- Windows 原生：只有显式设置 `WORD_MATCH_ALLOW_ONLINE_DOWNLOAD=1` 才允许在线下载 Node.js
- Windows WSL：默认禁用，只有显式设置 `WORD_MATCH_ALLOW_WSL_ONLINE=1` 才允许走联网方案

### 2. 应用离线能不能启动？

可以。

当前项目目录里已经有本地 `storage/dictionary.json`、`storage/packs.json` 等数据文件，安装、启动、停止本身不依赖外网。

### 3. 哪些功能仍然可能访问外网？

应用运行后的“在线更新词库”这类功能仍然可能访问远端接口，但这不影响安装、启动、停止本身。

### 4. 如果启动失败，先看哪里？

优先看这两个文件：

- `.runtime/server.stdout.log`
- `.runtime/server.stderr.log`

如果端口冲突，启动脚本也会在终端或窗口里直接提示已切换到哪个端口。
