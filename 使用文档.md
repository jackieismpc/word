# 单词对对碰使用文档

本文档只说明如何使用项目自带脚本完成安装、启动和停止。

## Mac

进入项目目录后执行：

```bash
chmod +x ./install-word-match.sh ./start-word-match.sh ./stop-word-match.sh
./install-word-match.sh
./start-word-match.sh
```

停止服务：

```bash
./stop-word-match.sh
```

说明：

- `install-word-match.sh` 会自动检查 Node.js
- 如果系统里没有可用的 Node.js 18+，脚本会自动下载项目本地运行时到 `.runtime/node`
- 本项目不依赖 npm、pnpm 或额外 `node_modules`

## Ubuntu

进入项目目录后执行：

```bash
chmod +x ./install-word-match.sh ./start-word-match.sh ./stop-word-match.sh
./install-word-match.sh
./start-word-match.sh
```

停止服务：

```bash
./stop-word-match.sh
```

说明：

- Ubuntu 与 Mac 共用同一套脚本
- 如果系统里没有可用的 Node.js 18+，脚本会自动下载项目本地运行时到 `.runtime/node`
- 服务日志默认写入 `.runtime/server.stdout.log` 和 `.runtime/server.stderr.log`

## Windows

### 方案一：Windows 原生脚本

在资源管理器中按顺序双击：

1. `install-word-match.cmd`
2. `start-word-match.cmd`

停止服务时双击：

1. `stop-word-match.cmd`

说明：

- 脚本会自动查找系统里的 Node.js
- 如果没有可用的 Node.js 18+，会自动下载项目本地运行时到 `.runtime\node`
- 本项目不依赖 npm、pnpm 或额外 `node_modules`

### 方案二：Windows WSL2 Ubuntu 保底方案

如果 Windows 原生环境启动不稳定，按顺序双击：

1. `install-word-match-wsl.cmd`
2. `start-word-match-wsl.cmd`

停止服务时双击：

1. `stop-word-match-wsl.cmd`

说明：

- `install-word-match-wsl.cmd` 会自动启用 WSL2、安装 Ubuntu，并在 Ubuntu 中准备运行环境
- 这个脚本首次运行可能会触发管理员授权；如果 Windows 提示需要重启，重启后再执行一次 `install-word-match-wsl.cmd`
- `start-word-match-wsl.cmd` 会自动在 Ubuntu 中安装缺失依赖并启动服务，然后在 Windows 浏览器打开地址
